# 基本設計

Electric Sheep は、日々の思い出・感情・写真・音声を長期的に残し、将来の自分や家族が忘れていた記憶にたどり着けるライフログサービスです。

このディレクトリには、バックエンド実装の前提となる基本設計をまとめます。

## ドキュメント構成

- [system_overview.md](./system_overview.md): システム概要、スコープ、利用者、非機能要件
- [architecture.md](./architecture.md): システム構成、コンポーネント責務、処理フロー
- [api_spec.md](./api_spec.md): REST API 仕様、認証、共通エラー形式
- [db_schema.md](./db_schema.md): エンティティ、テーブル設計、制約、マイグレーション方針

## 設計方針

- 個人的な記憶を扱うため、認証・認可・公開範囲の制御を重視する。
- 100年使うことを想定し、データ構造は移行しやすく、JSONとメディアファイルをまとめてエクスポートできる形にする。
- MVPでは認証、投稿、閲覧、共有、生成AI検索・要約、通知、エクスポートまでを対象にする。
- バックエンドは Go + Gin を前提とし、API は `/v1` でバージョニングする。
- DB は Neon PostgreSQL を利用する。標準的な PostgreSQL 機能を中心に使い、将来的に RDS や Cloud SQL などへ移行しやすい設計にする。
